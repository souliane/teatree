# Memory Consolidation — the idle-time "dreaming" pass

> BLUEPRINT §4 appendix. The idle-time memory-consolidation ("dreaming") data layer and phase mechanics, split out of the top-level BLUEPRINT so §4 keeps the architecture and this file holds the depth. Cited as `BLUEPRINT §4` from `teatree.loops.dream`.

**Memory-consolidation data layer ([#1933](https://github.com/souliane/teatree/issues/1933)).** Three durable rows back the idle-time "dreaming" pass that clusters session feedback and distils each cluster into one imperative rule. `ConsolidatedMemory` is the consolidation ledger (the DB row is canonical state + audit trail; a rendered topic file is the output), idempotency-anchored on a sha256 `cluster_key` and owning its status ladder `CANDIDATE → VERIFIED → PROMOTED` (plus `SUPERSEDED`/`EXPIRED`); it refuses an uncited rule, refuses to silently expire a BINDING row, and gates index-line pruning on a terminal status with a recorded durable home.

**Pass 2 — memory→fix-and-merge ([#2426](https://github.com/souliane/teatree/issues/2426), [#2663](https://github.com/souliane/teatree/issues/2663)).** A `disposition` ladder (`UNTRIAGED → USER_SPECIFIC_KEEP | (CORE_GAP_NEEDS_TICKET → TICKETED → RESOLVED_RETIRED)`) makes the ledger a queue that DRAINS, not a pile that grows: a generic/teatree-core memory is a confession that core has a workflow gap, so `teatree.loops.dream.promote_memory` triages each untriaged row (injected `MemoryClassifier`, default the `durable_destination`-hint one — a home under teatree's own skills/source/scripts/BLUEPRINT is core-generic, anything else stays user-specific memory). The promote verb no longer files a fresh `needs-triage` issue per gap — those PILED UP because the `IssueImplementerScanner` SKIPs `needs-triage`. Instead each core gap is driven to a MERGED fix via `teatree.loops.dream.umbrella_ledger`: a CHECKBOX is upserted under the standing umbrella issue ([#2663](https://github.com/souliane/teatree/issues/2663) — reused daily, NEVER closed), keyed on an invisible `<!-- dream-gap <cluster_key> -->` marker so the same gap never double-adds (banned-term + bare-reference scanned so the public-repo title never leaks), and the fix is SCHEDULED for a coding agent by reusing `Ticket.schedule_coding()` — the PR merges through the SAME §17.4 keystone gated by the overlay's autonomy. The new `CodeHostBackend.update_issue(issue_url, body)` extension method edits the umbrella body in place (GitHub `PATCH .../issues/<n>`, GitLab `PUT .../issues/<iid>`). When a gap's fix Ticket reaches MERGED, `reconcile_merged_gaps` CHECKS its umbrella checkbox and retires the prose (`RESOLVED_RETIRED`, archived) through the existing `retire_resolved_memories` (driven off the Ticket's authoritative MERGED state via an injected `is_resolved` predicate, not a fragile forge re-read of a `/pull/<n>` URL). **No new model:** an in-flight gap is an existing `Ticket` row (gap/memory/umbrella linkage in `extra.dream_gap_key`/`dream_memory_cluster_key`/`dream_umbrella_url`) + its `ConsolidatedMemory` entry; the #2663 checkbox is the durable cross-night state. A BINDING row is never retired. Pass 2 schedules the fix (durable, reversible) but never hand-implements; it is **default OFF** behind the `[loops.dream] memory_promote` / `T3_DREAM_MEMORY_PROMOTE` toggle, so an unflagged pass never triages, upserts, or schedules.

**Phase 3c — the Instruction-Compliance Accountant ([#2663](https://github.com/souliane/teatree/issues/2663)).** Instruction compliance is the root KPI: a rule was PRESENT/AVAILABLE (a durable memory, a loaded-skill rule, a CLAUDE.md clause, a system gate, or an explicit in-session directive) and the agent acted against it. `teatree.loops.dream.compliance` mines one pass's extract (reusing `transcript_extract.looks_like_user_correction` as the ground-truth signal) into typed `ComplianceFinding`s — a correction overlapping a memory-backed rule already on disk is a RECURRENCE (`rule_source=MEMORY`), one with no backing memory a first-occurrence `IN_SESSION` violation — and persists one `InstructionComplianceSnapshot` per pass (`instructions_observed`/`violations`/`compliance_rate`/`recurrences_count`) plus one `InstructionComplianceRecord` audit row per violation. **The binding escalation rule:** a recurrence (a rule that already had a durable memory, violated AGAIN) must escalate to a gate or an eval, NEVER another memory — `escalate_recurrences` drives ONE umbrella checkbox + a scheduled fix per recurring rule (deduped on the `compliance-recurrence-<rule_identity>` gap key) via the same `umbrella_ledger.promote_gap` path Pass 2 uses, the checkbox title prescribing a structural fix (a PreToolUse/Stop gate, a config self-check, or an anti-vacuous `under_load` eval), never more prose and no longer a fresh `needs-triage` issue; and `write_clusters` reclassifies a MEMORY_ONLY cluster whose rule already shows a recurrence off its memory destination to a core-gap home rather than re-promoting another memory. Compliance MEASUREMENT runs on **every** pass (tick + run), **default ON** behind the `[loops.dream] compliance_measure` / `T3_DREAM_COMPLIANCE_MEASURE` kill-switch (fault-isolated, persists a snapshot, never files); ESCALATION stays **default OFF**, gated by `t3 dream run --full` AND the `[loops.dream] compliance_escalate` / `T3_DREAM_COMPLIANCE_ESCALATE` toggle; `t3 dream compliance show` prints the latest snapshot (rate + recurrence count + open escalations).

**Phase 3d — the Automatable-Ask Promoter ([#2663](https://github.com/souliane/teatree/issues/2663)).** The structural "improve-with-new-stuff" sibling of the compliance accountant (the "fix-issues" half): dreaming detects recurring MANUAL user asks t3 could automate and promotes each as a fix-and-merge so the user "gets out of the loop", expressed ENTIRELY within the memory verbs (consolidate → merge → promote → retire), **no new model**. `transcript_extract.looks_like_user_ask` is the keyword-blind keeper (the sibling of `looks_like_user_correction`): a USER turn reading like a directive/request (imperative cues "can you"/"please"/"let's"/"set up", or operational-ACTION cues "hotfix"/"asap"/"rollback" — bare incident-state words "production"/"broken"/"blocker"/"wedged" are excluded, [#2732](https://github.com/souliane/teatree/issues/2732), as they describe a situation not a request); it rides `high_signal_lines` (and the narrow `user_ask_lines` sibling) so repeated asks reach the distiller and cluster into `ConsolidatedMemory` ask-clusters (same `cluster_key` upsert dedups/merges for free). `teatree.loops.dream.automation_ask` classifies each ask-cluster Bucket A (`EXISTING_GAP` — an existing loop/skill SHOULD have handled it; the injected `AUTOMATION_CATALOG` of loop+skill names lets the classifier name the real mechanism, and the gap prescribes fixing its trigger/coverage/surfacing) or Bucket B (`NEW_WORKFLOW` — no automation exists; prescribe a new loop/skill/gate, canonical example a hotfix lane), via an injected `AskClassifier` seam (default the deterministic catalog classifier, so the phase runs LLM-free). Each GROUNDED ask gap (the verbatim cited snippet must appear in the transcript — reusing `engine.cluster_is_grounded`) routes through the SAME `umbrella_ledger.promote_gap` path Pass 2 / Phase 3c use — a deduped umbrella checkbox + a scheduled coding fix, the title carrying the Bucket-A/B framing and a `dream-automation-gap` lineage tag; retirement reuses `reconcile_merged_gaps` (the ask memory retires off the gap-fix Ticket's MERGED state). The phase runs under `t3 dream run --full`, **default OFF** behind the `[loops.dream] automation_asks` / `T3_DREAM_AUTOMATION_ASKS` toggle; an unflagged pass never promotes an ask. `DreamQaProbe` is the retention/interference/monotonicity QA corpus, now POPULATED and replayed by the **§ 4 acceptance gates** ([#2545](https://github.com/souliane/teatree/issues/2545), `teatree.loops.dream.gates`). The cron snapshots each memory dir BEFORE and AFTER the file-side phases and runs all seven gates: **(a) retention** (every probe answerable pre-pass is still answerable post-pass), **(b) interference** (the prior-session probe pass-rate does not regress), **(c) consolidation-actually-happened** (net size reduced OR schema count grew OR clusters were recorded, AND every pruned index line has a confirmed durable home), **(d) index-budget** (`MEMORY.md` back under the real ~24 KB SESSION-LOAD byte-truncation point, not a 10x regression alarm — so an over-budget index trips gate (d) RED while still recoverable; [#2723](https://github.com/souliane/teatree/issues/2723)), **(e) monotonicity** (two passes over a stable corpus do not lower the pass-rate), **(f) no-loss audit** (every archived entry records a source + a destination that exists), and **(g) compliance-non-regression** ([#2663](https://github.com/souliane/teatree/issues/2663) — a recurrence remediated with another memory instead of a gate/eval FAILS the pass; an escalated recurrence passes). A probe is seeded per memory (keyed on a signature line) and replayed via an injectable, deterministic LLM-free answerer; the probe corpus is persisted (idempotent on `probe_key`, accumulating pass/run counts) so the next pass has a baseline. A **failing gate marks the pass attempted-not-succeeded** (staleness keeps firing) — a lossy / delete-only / no-op consolidation is caught, never stamped success. Each gate has an anti-vacuous test proven in both directions, and the command's gate-gating is mutation-proven (ignoring the verdict turns the wiring tests RED). `DreamRunMarker` is the staleness marker keyed on the success timestamp (a run that keeps failing stays stale). `t3 dream run` (manual) and `t3 dream tick` (cadence-gated cron, ~04:00) take a `dream-tick` `LoopLease` and run the distillation engine (`teatree.loops.dream.engine.run_consolidation`), stamping `DreamRunMarker` only on a clean pass; `--dry-run` writes nothing. Engine v1 covers phases 1–3 (rank a bounded extract → an injected `Distiller` clusters by root cause → upsert by `cluster_key`, rejecting any uncited/hallucinated cluster and never overwriting a BINDING rule); the injected-distiller seam keeps the pass testable, and an SDK failure propagates rather than laundering into a fake success. The distiller (and the eval synthesizer) authenticate their `claude` subprocess through the same Layer-2 `agent_harness_provider` credential child-env as the headless lane, resolved at global scope for the Task-less system pass, so an unresolvable credential fails the pass loud instead of surfacing as an `UNPARSABLE` empty result.

**Phases 4–6 are LIVE** ([#1933](https://github.com/souliane/teatree/issues/1933) § 6): the cron runs the memory-file phases (cross-link, merge, re-index, decay) over the discovered `~/.claude` memory dirs, each fault-isolated in its own try/except so one phase (or one dir) failing never crashes the tick or stops the others, and each LIVE by default behind its own two-layer kill-switch (`[loops.dream] <phase>` toml / `T3_DREAM_<PHASE>` env, first match wins). The file-side phases run independently of the transcript extract, so they fire even on a 0-transcript-member pass (which stays attempted-not-succeeded for consolidation, but still re-indexes/decays the on-disk memory set; [#2547](https://github.com/souliane/teatree/issues/2547)).

**Phase 4** (`teatree.loops.dream.cross_link`) appends symmetric `[[name]]` wiki-links between related memories — related when their significant-token Jaccard clears a deterministic floor (no model) — and is idempotent (a re-run on an unchanged set adds nothing).

**Phase 4b** (`teatree.loops.dream.merge`, [#2723](https://github.com/souliane/teatree/issues/2723)) is the on-disk MERGE verb cross-link was missing: it collapses a near-duplicate PAIR (body-token Jaccard ≥ 0.85 AND same frontmatter `type`/`name` family — far above cross-link's floor) into the HIGHER-WEIGHT survivor (a BINDING rule outranks a plain restatement, so binding doctrine is never the side absorbed), appends the absorbed file's distinct lines with a provenance header, and ARCHIVES the absorbed file via `decay._archive_one` (moved, never deleted) so the re-index drops its pointer in the same pass.

**Decision-3 (binding):** two BINDING near-duplicates are NEVER auto-merged — they are cross-linked and a deduped `dream-binding-reconcile` ticket is filed via the existing `promote_memory` filer (reusing its banned-term / bare-reference withholding) for human reconciliation. Phase 4b runs AFTER cross-link, BEFORE re-index, LIVE by default behind the `[loops.dream] merge` / `T3_DREAM_MERGE` kill-switch, and is fault-isolated. The whole file-side phase orchestration (4/4b/5/6 + the § 4 gates) is a composed `teatree.loops.dream.phase_runner.MemoryPhaseRunner`.

**Phase 5** (`teatree.loops.dream.reindex`) regenerates `MEMORY.md` as one tight line per memory — a single bare `name.md` pointer (no `[name.md](name.md)` href duplication) plus a ≤45-char brief hook, the WHOLE line capped at 130 chars so it stays under the gate-(d) load budget — deduped, filename-stable, content stays in the topic file, byte-identical on a no-change re-run.

**Phase 6** (`teatree.loops.dream.decay`) ages a memory out by ARCHIVING it (moved to `<memory_dir>/archive/` with a provenance header, never deleted) — gated by a NON-VACUOUS retention guard that retains a memory when ANY of: it is fresh (mtime within the window), it is still referenced (another live memory or the index `[[name]]`-links it), OR — the **transfer-before-prune rail** ([#2546](https://github.com/souliane/teatree/issues/2546), #1933 § 2) — its lesson has NO confirmed durable home in the `ConsolidatedMemory` ledger. Only a memory that is old AND unreferenced AND durably homed is archived; a guard-disabled probe proves each guard has teeth. The §2 rail — *"delete an index line only after the fact has a confirmed durable home in a topic file"* — is applied to the topic file itself: an injected `HomeResolver` seam keeps the file-side mechanics DB-free under test, while the production `ledger_durable_home_resolver` reads `ConsolidatedMemory.objects.prunable()` (the index-pruner's terminal-status + recorded-`durable_destination` surface) once per pass and maps a topic file to a row by path membership in `source_files` OR by its name appearing in a `durable_destination`.

**Budget RETIRE tier ([#2723](https://github.com/souliane/teatree/issues/2723)).** The `prunable()` home-rail is structurally EMPTY for the hand-authored corpus (0 ledger rows reference on-disk memories), so it can never archive the files that bloat the index. A second, ledger-INDEPENDENT tier (opt-in via `DecayPolicy(budget_tier=BudgetTier())`, enabled in the cron) fires ONLY when `MEMORY.md` is over the gate-(d) load budget and archives the LOWEST-`_signal_score` files first — scoring each by user/BINDING weight, inbound `[[name]]` links (+40 each), a LOGICAL `lesson_updated` recency clock (NOT raw `st_mtime`, so a cross-link / re-index rewrite does not reset it), and a per-type floor — only as many as it takes to bring the projected hot index back under budget. Referenced entries are NOT hard-retained ([#2753](https://github.com/souliane/teatree/issues/2753)): the cross-link phase runs BEFORE decay and references most of the corpus, so a hard skip floored the tier above budget and it could never converge; the +40-per-link signal instead ranks referenced entries HIGHER so they are archived LAST, only when the budget genuinely forces it. Every archived entry stays restorable in `archive/` (full body + provenance) and recall-able via the cold `MEMORY_ARCHIVE.md` signature, so a now-dangling `[[link]]` in a surviving body is cosmetic, not data loss (the hot index uses bare `- name.md` pointers, which never dangle); the conservative stale/ledger tier keeps its reference skip. The cron re-indexes AFTER this decay so `MEMORY.md` falls back under the gate-(d) budget in the SAME pass. All phases accept an explicit `memory_dir` and never touch the real `~/.claude` under test.

**Cold-tier RECALL ([#2746](https://github.com/souliane/teatree/issues/2746)).** The budget tier moves the lowest-signal memories — ~130 BINDING/Non-Negotiable rules of a ~693-entry corpus — to the NOT-session-loaded cold `MEMORY_ARCHIVE.md`, so they stop influencing behaviour. A `UserPromptSubmit` hook (`hooks/scripts/memory_recall.py`) closes that gap: the DB-free, stdlib-only pure core `teatree.loops.dream.recall` scores each cold-index line against the prompt's tokens (ambient `<system-reminder>`/`<command-*>` blocks stripped FIRST so an injected `MEMORY.md`/`CLAUDE.md` dump can never self-match), applies a 2-match relevance floor BEFORE any BINDING(+3)/`user_`(+2) boost (an irrelevant BINDING rule is never surfaced on one incidental token), dedupes against the hot index (pointer name + a renamed-file signature-substring guard), and injects the top ≤5 hits (≤1500 bytes, each line ≤200 chars) for that one turn; it is registered LAST on `UserPromptSubmit` so it never delays skill-loading, imports the core without `django.setup()`, and is crash-proof/silent on any degrade. Off-switch `[teatree] memory_recall_enabled` (default ON; self-rescue `t3 <overlay> gate memory-recall disable`), and `t3 <overlay> memory recall <query>` is the manual-lookup CLI. The recall quality and the §4 retention probe share ONE frontmatter-aware extractor `reindex.signature_text` ([#2746](https://github.com/souliane/teatree/issues/2746) nit-4) — preferring a memory's frontmatter `description:`/`summary:` over the weak `node_type: memory` body line — routed through by `reindex._summary_for` (hot index) and `gates._signature_line` (cold index + probe), so the hot index, cold index, and retention probe derive the SAME signature. The shared `_teatree_bool_setting` reader is extracted to the `hooks/scripts/teatree_settings.py` leaf so a gate leaf reads its kill-switch without a `hook_router`↔leaf import cycle.

**Phase 3b** ([#2346](https://github.com/souliane/teatree/issues/2346)) closes the drift → live-eval loop: `teatree.loops.dream.eval_proposer` maps each grounded cluster to an eval CANDIDATE in a JSONL review queue, then `teatree.loops.dream.promote` PROMOTES it to a real `under_load` scenario (`evals/scenarios/promoted_drift.yaml` + `_pass`/`_fail` replay fixtures) — the artifacts the deterministic replay gate and the weekly metered lane run unchanged. The nightly `tick` requests proposals **LIVE by default** (the `[loops.dream] propose_evals` / `T3_DREAM_PROPOSE_EVALS` kill-switch flips it off). Promotion is auto but gated by a **non-bypassable anti-vacuity guard** (`promote.guard_can_fail`): it runs the real grader (`teatree.eval.report.evaluate`) against a synthesised drift-recommitting transcript and promotes only when that verdict is FAIL (the matchers have teeth) AND a compliant transcript grades PASS (not a tautology); a candidate whose grader cannot fail is rejected — no file written. The guard is deterministic; live metered grading of a promoted scenario lands with the weekly Agent-SDK lane.

**Phase 3b LLM derivation** ([#2447](https://github.com/souliane/teatree/issues/2447)) is the richer, **default-OFF** follow-up to the fixed-matcher `promote`: `teatree.loops.dream.llm_eval_proposer` runs an injected LLM synthesizer that turns a grounded candidate plus its cited transcript slice into a COMPLETE `under_load` `EvalSpec` — a pollution preamble saturated to the documented ~28k-char envelope floor, discriminating positive + negative matchers, and a judge rubric — parsed through the real loader and gated by a **candidate-derived** teeth check (`teatree.loops.dream._teeth_check.teeth_check_against_candidate`): it synthesises the `_fail`/`_pass` transcripts FROM the candidate's OWN cited drift and compliant tool-call shapes and grades both through the suite's real grader (`teatree.eval.report.evaluate`), DROPPING a spec whose matchers do not reject the candidate's own cited drift (or reject its compliant shape as a tautology) — proving the matchers target the SPECIFIC drift the candidate describes, not `promote`'s fixed session.py-edit transcripts. Even a proven spec is never autonomously committed to `evals/scenarios`: `stage_derived_evals` writes a staging `derived_evals.yaml` a human / standing core-maker ratifies via a PR. The phase is OFF unless the `[loops.dream] derive_evals` / `T3_DREAM_DERIVE_EVALS` toggle is set, so an unflagged pass is byte-identical. The `dream` mini-loop is registered `off_live_tick` (driven only by its own cron); `t3 doctor` surfaces the 48h staleness alarm.
