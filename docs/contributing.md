# Contributing

Notes for working on TeaTree itself. The canonical, always-current rules live in
[`AGENTS.md`](https://github.com/souliane/teatree/blob/main/AGENTS.md) and
[`BLUEPRINT.md`](https://github.com/souliane/teatree/blob/main/BLUEPRINT.md) — this page is a short
orientation, not a second source of truth.

## Local setup

```bash
git clone https://github.com/souliane/teatree && cd teatree
uv sync
uv tool install --editable . && t3 setup
```

## Day-to-day commands

```bash
uv run pytest                 # full suite, parallel (-n auto), no coverage — fast default
bash dev/test-cov.sh          # coverage lane: --cov --doctest-modules, 93% floor (CI parity)
uv run ruff check             # lint
uv run ruff format            # format
prek run --all-files          # all pre-commit hooks
```

## Worktree-first

Changes are made in a dedicated worktree per ticket, not on the main clone's default branch.
A pre-commit hook refuses commits made directly in the main clone on a non-default branch, so
create the worktree before touching files.

## Documentation

The reference pages under **Reference** are generated, not hand-written:

- CLI reference — regenerated by the `generate-cli-reference` pre-commit hook from the Typer
  command tree.
- Overlay extension points, skill delegation matrix, and the skills catalogue — regenerated by
  `manage.py generate_all_docs` from `OverlayBase` type hints, the skill map, and
  `skills/*/SKILL.md` frontmatter.

```bash
uv run python manage.py generate_all_docs
```

A CI job (`docs-drift`) regenerates these and fails if the committed output is stale, so the
site never drifts from the code. If that job is red, run the command above and commit the
result.

## Building the site locally

```bash
uv sync --group docs
uv run mkdocs serve     # live preview on http://127.0.0.1:8000
uv run mkdocs build     # strict static build
```

The site deploys to GitHub Pages from `main` via the `docs` workflow.
