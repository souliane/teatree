# Slack DM Format — read when composing the post-scan DM

Terse. One idea per line. Blank line between blocks. Answer first.

## Standard format

```text
scanning-news — <YYYY-MM-DD today>

TLDR AI <issue date>:
  • <story title> — <1-line why interesting> <link>
  • <story title> — <1-line why interesting> <link>

Rundown AI <issue date>:
  • <story title> — <1-line why interesting> <link>

tickets filed:
  • souliane/teatree#<N> — <issue title>
  • souliane/teatree#<N> — <issue title>
```

## Zero-result format

```text
scanning-news — <YYYY-MM-DD>: 0 relevant items, 0 tickets.
```

## Partial-failure format

```text
scanning-news — <YYYY-MM-DD>

TLDR AI <issue date>:
  • <story title> — <1-line why interesting> <link>

Rundown AI: fetch failed (aggregator after retry)

tickets filed:
  • souliane/teatree#<N> — <issue title>
```

## Date-fallback format

When today's edition isn't published yet and the scan falls back to the latest:

```text
scanning-news — <YYYY-MM-DD today>  (TLDR AI not yet published today, scanned <fallback date>)
```

## Posting

You do NOT post this DM. You run shell-denied — you return the candidates in the
result envelope's `article_suggestions` field, and the loop posts ONE
batch-approval DM (via `notify_user`, self-DM → bot token) after it records them.
The format above is the shape that DM takes; it is here for reference, not for you
to send. No AI signature, no "generated by" trailer — the loop's egress enforces
this.
