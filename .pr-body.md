## Bug
- Boundary-agent gate demanded `run_in_background: true` on every Agent dispatch.
- Current Claude Code Agent tool omits `run_in_background` (async by default; #1646 signal removed in a later CC release).
- Result: absent field read as foreground → EVERY main-agent Agent dispatch false-blocked.

## Fix
- Deny only an EXPLICIT `run_in_background: False`.
- Absent field = background dispatch = allowed.
- Gate's purpose (stop a blocking foreground dispatch) preserved.
