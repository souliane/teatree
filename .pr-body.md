Addresses owner directive: auto-merge same-repo trusted PRs (strict fork-holds) + move sweep to ship loop so tickets advance one-by-one under autonomous_away.

## What & why

Two coupled changes toward safe autonomous merge, plus one safety complement:

1. **Same-repo-vs-fork merge provenance (strict fork-holds).** A PR whose head branch lives in a **fork / cross-repo** ALWAYS requires human approval — even from a trusted author — because a fork PR is attacker-controllable code proposing itself for auto-merge. A **same-repo** head branch is the operator's own push and is trusted (author-independent). Unreported provenance fails closed to the existing identity+visibility author check. The two MERGE gates (the sweep rung + the keystone) adopt `classify_pr_provenance`; the reviewing scanners keep the pure-identity `classify_author`. Resolves via `CodeHostQuery.pr_same_repo()` — GitHub `isCrossRepository`, GitLab `source_project_id == target_project_id` — so overlay MRs cross the same gate. Defence-in-depth: the gate also fires inside the shared `execute_bound_merge` chokepoint, so the solo-overlay `merge_pr_squash_bound` bypass crosses it too.

2. **Move the PR sweep from the review loop to the ship loop.** The review loop is `colleague_facing` and is skipped under `autonomous_away`, which starved the auto-merge path exactly when the operator is away. Ship is enabled and ticks every 5m and its seed already claims the keystone merge. The solo review-arm widens to `pr_authored_by_self(...) or same_repo is True` so same-repo bot PRs gain the cold-review the sweep merges on.

3. **Widen the substrate hold to the factory's self-governance seams** (safety complement). Under `autonomy=full` a non-substrate PR auto-merges on agent-only cold review. Changes to the trust classifier (`author_trust.py`), the intake gate (`issue_implementer.py` / `scanner_factories.py`), and the safety hooks (`hooks/`) now classify as substrate ⇒ ping-and-hold for the owner. The factory cannot loosen its own guardrails unattended.

This PR touches `src/teatree/core/merge/` and `author_trust.py` → it is substrate → the keystone holds it for the owner's explicit merge sign-off (expected; that safety property is preserved).
